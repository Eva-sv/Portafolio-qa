Projects

Web Page – Manual QA

I performed Quality Assurance for an already developed web application, working as part of a team under Agile Scrum methodologies. Work management was carried out through weekly sprints using Jira, activily participating in Sprint Planning, daily meetings and Sprint Review at the end of each sprint.

I designed tests for user stories following the BDD methodology, based on the defined acceptance criteria. Gherkin language scenarios were created to provided readable and understandable tests for the entire team, managing them with AssertThat to maintain traceability between requeriments,tests and results within Jira. Tests were executed per user story and results were recorded in web and mobile environments.

During each sprint functional testing was performed and, at the end of of the sprint, exploratory testing was conducted to identify defects not covered by the defined test cases. Detected bugs were reported in Jira, linking them to the corresponding user story to ensure proper traceability between requirements, tests and defects.

At the end of each sprint, the release was prepared in Jira and the related delivery information was documented in Confluence, including the complete test plan documentation, execution results and the product quality status.

After the release, the main application flows were identified and regresion tests were automated using Phyton,Playwright and Pytest. These tests were integrated into a continuous integration workflow with GitHub Actions,configured to automatically run regression test weekly, aligned with the end of each sprint.

Portfolio QA – Web Developer and QA in APIs Testing

I developed a web portfolio using HTML5, CSS3 and JavaScript, integrating a form connected to an API generated with Supabase. The data base tables were designed and SQL queries were performed for inserting and retrieving test data from the form.

I carried out API testing using Postman, creating tests for GET (retrieve), POST (create), PATCH (update) and DELETE (remove) requests. Test were implemented to validate positive, negative and authentication response codes, as well as the correct structure and content of responses.

Additionally, a continuous integration workflow was implemented with GitHub Actions, configuring Newman to automatically run the Postman tests whenever changes are made to the repository.
